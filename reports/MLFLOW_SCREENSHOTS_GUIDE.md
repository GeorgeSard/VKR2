# Инструкция: как поднять MLflow UI и сделать скрины для отчёта

## Шаг 1. Запустить MLflow UI

Открой терминал, перейди в корень проекта и выполни **одну команду**:

```bash
cd /Users/georgij/Projects/ВКР2
source .venv/bin/activate
mlflow ui --backend-store-uri "file:$PWD/mlruns" --host 127.0.0.1 --port 5000
```

После этого терминал «зависнет» — это нормально, MLflow держит сервер.
В браузере открой:

> **http://127.0.0.1:5000**

Чтобы потом остановить — `Ctrl+C` в том же терминале.

### Если порт 5000 занят

```bash
lsof -i :5000          # посмотреть, что занимает
mlflow ui --backend-store-uri "file:$PWD/mlruns" --host 127.0.0.1 --port 5001
```

Открыть на http://127.0.0.1:5001.

### Если хочется, чтобы UI работал в фоне (терминал свободен)

```bash
nohup mlflow ui --backend-store-uri "file:$PWD/mlruns" --host 127.0.0.1 --port 5000 > /tmp/mlflow_ui.log 2>&1 &
```

Остановить: `pkill -f "mlflow ui"`.

## Шаг 3. Скрины для отчёта — что конкретно снимать

Список упорядочен по тому, как они логически идут в главе 3 ВКР.

### Скрин 1. Обзор всех 13 runs списком

**Что показывает:** масштаб серии экспериментов.

**Как сделать:** просто открыть страницу эксперимента. По умолчанию
видны последние ~10 runs — внизу страницы лимит можно поднять до 25.
Снять скрин всей таблицы.

**Куда в отчёт:** в начало главы 3, как «вот что мы делали».

---

### Скрин 2. Прогресс binary-серии — параллельные координаты

**Что показывает:** как метрики растут от Run #1 (logreg basic) к
Run #6 (xgb optuna).

**Как сделать:**
1. В строке поиска вверху написать:

   ```
   tags.task = "delay_binary"
   ```

   Останутся только 8 binary runs.
2. Выделить **все 8** галочками слева.
3. Нажать кнопку **Compare** сверху.
4. На странице сравнения переключить вид на **Parallel Coordinates Plot**
   (вкладка/кнопка вверху правее графика).
5. В выборе осей слева добавить: `feature_set`, `model`, `f1`, `roc_auc`.
6. Снять скрин — будет видно, как линии (= runs) идут по осям и где
   достигают максимума.

**Куда в отчёт:** в раздел про дельты Δ1-Δ5 binary-серии.

### Скрин 3. Лидер binary — карточка Run #6

**Что показывает:** конкретные best params + метрики финальной
binary-модели + артефакты.

**Как сделать:**
1. В таблице runs найти строку с `Run Name = optuna-xgboost-with_weather`
   (это Run #6, run_id начинается на `4227cd7b`).
2. Клик по имени run.
3. Снять три скрина:
   - вкладка **Parameters** — видно все best_* params от Optuna
   - вкладка **Metrics** — F1=0.630, ROC-AUC=0.839 и т.д.
   - вкладка **Artifacts** — слева дерево, открыть `best_xgboost_params.yaml`
     прямо в UI

**Куда в отчёт:** в раздел про финальную binary-модель.

### Скрин 4. Прогресс cause-серии C1-C5

**Что показывает:** параллельный путь второй головы — от макро F1

## Шаг 4. Альтернатива — экспорт таблицы в CSV

Если нужно вставить именно таблицу с метриками (не скрин), MLflow
позволяет экспортировать выбранные runs:

1. В таблице runs выделить нужные галочками.
2. Кнопка **Download CSV** сверху справа от таблицы.
3. Открыть CSV в Excel и форматировать как угодно.

## Полезные фильтры в строке поиска MLflow

```
tags.task = "delay_binary"            # только binary серия
tags.task = "delay_cause"             # только cause серия
tags.model = "xgboost"                # только xgboost runs
tags.feature_set = "with_weather"     # только финальный набор фичей
tags.tuning = "optuna_tpe"            # только Optuna-тюнинг runs
metrics.f1 > 0.6                      # только runs с F1 > 0.6
attributes.run_name LIKE "C%"         # только cause runs (по имени)
```

Фильтры комбинируются через `AND`:

```
tags.task = "delay_binary" AND metrics.f1 > 0.6
```

## Если что-то сломалось

| Симптом | Что делать |
|---|---|
| `mlflow ui` ругается «no module mlflow» | Не активирован venv. Запусти `source .venv/bin/activate` |
| Страница пустая, экспериментов нет | `ls mlruns/` — должны быть каталоги. Если пусто — данные потерялись (но `git log` показывает что они должны быть локально) |
| Cannot connect to localhost:5000 | Mlflow упал. Запусти заново. Если опять падает — `lsof -i :5000` показать что мешает |
| Кириллица в Run Name отображается мусором | Проблема браузера. Открыть UI в Chrome/Safari, должно быть нормально |
