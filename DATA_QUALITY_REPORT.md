## 6. Сбалансированность (Balance)

> Распределение классов в задачах классификации.

### Задача A — Бинарная классификация задержки

| Класс | Доля |
|---|---|
| `is_departure_delayed_15m == 0` (не задержан) | 74.04% |
| `is_departure_delayed_15m == 1` (задержан) | 23.97% |

Ratio 3.1:1. Это **умеренный дисбаланс**, типичный для задач предсказания задержек. Стандартные методы работы с дисбалансом (`class_weight='balanced'`, SMOTE, фокальная функция потерь) применимы и являются предметом отдельного эксперимента в главе 3.

### Задача B — Классификация причины задержки (среди задержанных)

| Класс | Доля |
|---|---|
| `weather` | 34.89% |
| `carrier_operational` | 32.35% |
| `reactionary` | 15.72% |
| `airport_congestion` | 15.45% |
| `security` | 1.58% |

Ratio мажорного к минорному: 22:1. **Умеренный-высокий дисбаланс** для класса `security` — реалистичен (security-задержки редкие, но крупные). Требует careful evaluation: macro-F1, per-class recall.

**Вердикт:** ⚠️ Дисбаланс присутствует и реалистичен. Это не дефект, а особенность задачи — требует методически правильной обработки в главе 3.

## Рекомендации для главы 3 (научные эксперименты)

1. **Эксперимент по дисбалансу:** сравнить методы (no-rebalancing / class_weight / SMOTE / undersampling) на задаче A и задаче B.
2. **Эксперимент по concept drift:** обучить на 2023, валидировать на 2024, тестировать на 2025 — изучить деградацию метрик и предложить стратегию ре-обучения.
3. **Эксперимент по feature ablation:** систематически исключать группы признаков (погода / оперативные / временные / маршрутные) и измерять вклад каждой группы.
4. **Эксперимент по двухстадийной vs одностадийной классификации причины:** сравнить «один многоклассовый классификатор» против «бинарный → многоклассовый среди задержанных».
