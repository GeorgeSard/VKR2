# Data Dictionary — Flight Delays RU 2023–2025

Полный словарь признаков синтетического датасета задержек авиарейсов России.

> **🚨 КРИТИЧЕСКИ ВАЖНО:** Колонки имеют чёткое разделение на категории.
> Префикс `gt_` означает **ground truth — НЕ ПРИЗНАК**, использовать как фичу запрещено.
> Колонки в категории «🎯 ТАРГЕТ» — то, что мы предсказываем, тоже не фичи.
> Всё остальное — допустимые признаки для feature engineering.

## 🛑 GROUND TRUTH компоненты задержки (⛔ НЕ ИСПОЛЬЗОВАТЬ КАК ПРИЗНАКИ)

> Эти колонки — компоненты декомпозиции таргета. Они нужны **только** для:
> 1. вычисления `probable_delay_cause` (категория argmax),
> 2. научного анализа в главе 3 (валидация качества классификации причин),
> 3. отчёта качества данных.
>
> Использование любой из них как признака для модели = **утечка таргета** = методологическая ошибка.

| Колонка | Тип | Описание |
|---|---|---|
| `gt_carrier_delay_minutes` | int | Компонента: задержка по вине авиакомпании (техника, экипаж, оборот) |
| `gt_weather_delay_minutes` | int | Компонента: задержка по погодным причинам |
| `gt_airport_congestion_delay_minutes` | int | Компонента: задержка из-за загруженности аэропорта |
| `gt_reactionary_delay_minutes` | int | Компонента: реактивная (каскадная) задержка от опаздавшего входящего борта |
| `gt_security_delay_minutes` | int | Компонента: задержка по причинам безопасности (план «ковёр», досмотр, угроза) |

## Рекомендуемая схема использования

### Задача A — Бинарная классификация задержки
- **Таргет:** `is_departure_delayed_15m`
- **Признаки:** все из категорий ✅, никаких `gt_*`
- **Метрики:** F1 (positive class), ROC-AUC, PR-AUC, recall@90% precision, accuracy

### Задача A' — Регрессия задержки
- **Таргет:** `dep_delay_minutes`
- **Подмножество:** `cancellation_flag == 0`
- **Метрики:** MAE, RMSE, R²

### Задача B — Классификация причины задержки (мультикласс)
- **Таргет:** `probable_delay_cause`
- **Подмножество (рекомендуется двухстадийная схема):** только `is_departure_delayed_15m == 1` (исключая `none` и `cancelled`)
- **Метрики:** macro-F1, weighted-F1, per-class precision/recall, confusion matrix

### Рекомендуемое разбиение train/val/test (имитация продакшена)
- **Train:** `year == 2023`
- **Val:** `year == 2024`
- **Test (production simulation):** `year == 2025`

Это разбиение моделирует реальную ситуацию: обучаемся на прошлом, валидируемся на недавнем, оцениваем на «новом». В 2024–2025 в данных присутствует concept drift (рост security-задержек на московских аэропортах) — это проявление обучения в условиях смещения распределения, что является **отдельным сюжетом для главы 4**.
